Quick Notes — SDLC at a Glance

Waterfall: Fixed, well-defined requirements. Linear phases. Pro: simple/traceable. Con: costly changes.
V-Model: Waterfall + test phase mapped to each dev phase. For safety/quality critical.
Prototype (Throwaway): Build a mock to clarify unclear UI/requirements; discard prototype and build real system.
Evolutionary Prototyping: Start minimal working system; evolve into final product as needs change (great for new/growing orgs).
Incremental: Deliver in modules/increments (e.g., Profiles → Payroll → Leave). Requirements mostly known per module.
Iterative: Rework the same product in cycles adding depth/quality (v1, v2, v3…).
Spiral: Iterative + rigorous risk analysis each loop. For high-risk, complex, costly systems.
Agile (Scrum/Kanban): Short sprints, frequent change, continuous delivery & customer collaboration.
RAD (Rapid App Dev): Fast delivery via prototypes, user workshops, reusable components and tools. Broad reqs known; details evolve.
Tip: Fixed req → Waterfall/V-Model • Unclear UI → Prototype • Evolving needs → Evolutionary/Iterative • Risky/complex → Spiral • Deliver in parts → Incremental • Fast with user buy-in → Agile/RAD
