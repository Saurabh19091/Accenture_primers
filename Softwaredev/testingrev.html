Software Testing Guide

1. Software Testing Overview

Software testing is a process used to identify the correctness, completeness, and quality of software. Testing can be broadly classified into Static Testing and Dynamic Testing.

2. Static Testing

Static testing is performed without executing the code. It focuses on reviewing and analyzing the code, documents, and design.

  • Techniques: Code Reviews, Walkthroughs, Inspections
  • Purpose: Detect errors early, save time and cost.
  • Pros: Finds syntax and documentation errors, prevents defects before execution.
  • Cons: Cannot find runtime errors.

3. Dynamic Testing

Dynamic testing involves executing the code to check if the software behaves as expected.

  • Purpose: Detect runtime errors, logic errors, performance issues.
  • Types: Functional Testing, Non-functional Testing.
  • Examples: Testing login modules, stress testing servers.

Black Box Testing

  • Does not require code knowledge.
  • Focus: Inputs and outputs.
  • Used in: System testing, Acceptance testing.

White Box Testing

  • Requires knowledge of code.
  • Focus: Internal logic, loops, conditions.
  • Used in: Unit testing, Integration testing.

4. Types of Software Testing

Functional Testing

  • Unit Testing
  • Integration Testing
  • System Testing
  • Acceptance Testing

Non-Functional Testing

  • Performance Testing
  • Load Testing
  • Stress Testing
  • Usability Testing
  • Security Testing

Comparison Table: Black Box vs White Box

Aspect Black Box Testing White Box Testing
Knowledge of Code Not required Required
Focus Functionality (inputs/outputs) Internal code structure
Techniques Equivalence, Boundary, Decision Table Statement, Branch, Path, Loop
Typical Level System, Acceptance Unit, Integration
Pros User-focused, easy to perform Thorough, finds hidden bugs
Cons Limited code coverage Requires programming knowledge
